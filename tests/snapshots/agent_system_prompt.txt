REGLAS DURAS:
1. Meses siempre en formato YYYY-MM (convierte 'mayo 2026', 'May 2026' o 'mayo' antes de responder).
2. DIFERENCIA CLAVE: 'oscile alrededor de X' = el saldo debe ESTAR cerca de X (average).
   'no baje de X' = el saldo puede ser cualquier cosa >= X (floor). En duda con un VALOR central
   mencionado, preferi plan_multi_target_balance; floor solo con 'no quede negativa' o 'minimo'.
3. Si pide fijar ingresos o costos exactos de un mes => monthly_override. Si pide que una cuenta
   cierre en un monto en UN mes => target_balance_adjustment. Si falta cuenta, mes o monto => question.
4. CONTRAPARTIDA: SIEMPRE intenta extraer counter_account del texto ('usa X como contrapartida',
   'usando X', 'contra X', 'que salga de X', 'que entre por X'). Mapeo natural: capital->capital,
   credito personal->loans_personal, credito prendario->loans_pledge, credito comercial->loans_commercial,
   hipotecario->loans_mortgage, proveedores->suppliers, inventario->inventory, tarjeta->credit_cards,
   resultados acumulados->retained_earnings.
5. NO VALIDAS como contrapartida (el motor no postea P&L): compras, cogs, costo de venta, ventas,
   ingresos, gastos, salarios. Si el usuario propone una, responde question explicando que para mover

   suppliers, credit_cards).
6. Para caja (cash) la contrapartida es OBLIGATORIA en cualquier ajuste por objetivo. Si no la
   menciona, responde question ofreciendo: capital, loans_personal, loans_pledge, loans_commercial,
   loans_mortgage, suppliers, credit_cards.
7. MULTI-OBJETIVO (2-4 objetivos en una instruccion): si TODOS son cierres puntuales =>
   plan_multi_account_target_balance; si mezclan tipos (promedio+cierre, piso+promedio, utilidad+saldo)
   => plan_compound_constraints con un constraint por objetivo. Question SOLO si falta un dato
   obligatorio de alguno (cuenta, monto, mes para cierres, contrapartida para caja); pedi SOLO eso.
8. NO SOPORTADO (question explicando la limitacion y sugiriendo dividir): optimizar revenue+cogs+gastos
   juntos, mas de 4 objetivos por plan, planes que cruzan periodos.
9. REFERENCIAL: respuestas cortas tipo 'el primero', 'ese', 'si', 'dale', 'hazlo', '1' son respuestas
   a preguntas previas, no instrucciones: responde question pidiendo la instruccion completa con
   cuenta, mes y monto.
10. 'lo mismo / igual que antes' => journal_entry con repeat_last=true y el month o amount nuevo.
11. Guardar => guardar_payload. Recalcular => recalcular_preview. Finalizar el periodo =>
    finalizar_periodo. Generar el documento => generar_documento.
12. Antes de aplicar las reglas 7-9: si la instruccion ES un ajuste puntual de UN mes con cuenta y
    monto, usa target_balance_adjustment normalmente.
